# 我的知识库

> 欢迎来到你的个人知识库（PKB）。从这里开始，将一切知识纳入体系化管理。

PKB 是一个编译式个人知识库，采用三层架构组织：原始资料层（raw/）-> 结构化知识层（wiki/）-> 自动化规则层（skills/）。所有操作均由 AI 辅助完成，你只需关注知识本身。

## 快速开始

最简单的使用方式：在对话中输入 `/pkb` 加上你想入库的任何内容。

```
/pkb <文件路径 | URL | 问题>
```

系统会自动识别输入类型并执行对应操作：导入文件、采集网页、搜索知识库、或执行健康检查。全程自动，无需手动干预。

详细命令参考见：[[../COMMANDS|命令参考]]

## 知识库结构

| 层级 | 目录 | 说明 |
|------|------|------|
| 原始资料层 | `raw/` | 不可变的原始资料。网页采集包、论文 PDF、导入的文档等均存放于此。只增不删。 |
| 结构化知识层 | `wiki/` | LLM 维护的结构化知识。包含概念笔记、来源索引、项目笔记，通过双链互连。 |
| 自动化规则层 | `skills/` | Agent 自动化规则和命令。定义了 `/pkb`、`/save` 等操作的具体执行逻辑。 |

知识流向：`raw/` -> `wiki/` -> `skills/`

- **raw/**：知识素材的永久存档，保持原始形式不修改
- **wiki/**：从素材中提炼的原子化概念、文献地图、项目进展
- **skills/**：将重复性操作固化为可复用的自动化流程

各层详细说明见：[[wiki/index|知识索引]]

## 常用命令

| 命令 | 用途 |
|------|------|
| `/pkb <anything>` | 统一入口：自动识别并处理任何输入 |
| `/save "消息"` | 保存当前状态并提交到 Git |
| `/inbox` | 查看待处理的导入文件 |
| `/ask <问题>` | 从知识库中搜索并回答 |
| `/lint` | 运行知识库健康检查 |
| `/docs-update` | 自动检测并更新项目文档 |
| `/rollback` | 查看或回滚 Git 历史 |
| `/help` | 显示帮助信息 |

更多命令和详细用法见：[[../COMMANDS|命令参考]]
